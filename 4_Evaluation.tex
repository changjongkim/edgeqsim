\section{Evaluation}
\label{sec:eval}

\subsection{Evaluation Setup}

We evaluate \EdgeQuantum on a representative resource-constrained edge device to demonstrate its capability to perform large-scale quantum circuit simulation within a limited power and memory budget.
The target hardware is the NVIDIA Jetson Orin Nano Developer Kit (8\,GB).
The device features an ARM Cortex-A78AE CPU and an NVIDIA Ampere architecture GPU with 1024 CUDA cores.
The system is equipped with 8\,GB of LPDDR5 Unified Memory shared between the CPU and GPU.
For secondary storage, we utilize a 256\,GB Samsung PM9A1 NVMe SSD connected via PCIe Gen3 x4, which delivers sequential read/write throughput of approximately 2.6\,GB/s and 1.8\,GB/s, respectively.
The entire system operates under a strict 15W power envelope.

We evaluate \EdgeQuantum using six representative quantum circuits: Quantum Volume (QV), Variational Quantum Classifier (VQC), Quantum Support Vector Machine (QSVM), Random, GHZ, and Variational Quantum Eigensolver (VQE).
We compare \EdgeQuantum with \textit{BMQSim-like}, a baseline representing a naive offloading approach that utilizes blocking I/O without the 4-buffer pipeline optimization.
Both schemes utilize the identical \textit{cuStateVec} backend for GPU execution, ensuring that performance differences are solely attributable to the I/O architecture.

\subsection{Performance Analysis: 29--34 Qubits}
\label{sec:perf_analysis}

Figure~\ref{fig:speedup_34q} and Table~\ref{tab:main_results} compare the simulation time of \EdgeQuantum against the \textit{BMQSim-like} baseline across state vector sizes ranging from 4\,GB (29 qubits) to 128\,GB (34 qubits).
This range represents the "Tiered Memory" regime where the state vector exceeds the 8\,GB physical memory of the device but fits within the SSD capacity without compression.

\noindent\textbf{Overall Speedup.}
\EdgeQuantum consistently outperforms the blocking I/O baseline across all circuit types and qubit counts.
The speedup becomes more pronounced as the problem size increases.
For example, at 29 qubits (4\,GB), where the OS page cache still provides some benefit, the speedup is modest (approx. 1.09--1.18$\times$).
However, at 33--34 qubits (64--128\,GB), where I/O contention becomes the dominant bottleneck, \EdgeQuantum achieves significant gains.
Notably, for the QSVM circuit at 33 qubits, \EdgeQuantum reduces the runtime from 430.7s to 291.4s, achieving a peak speedup of \textbf{1.48$\times$}.
Similarly, for the VQC circuit at 31 qubits, \EdgeQuantum achieves a \textbf{1.45$\times$} speedup (254.7s vs. 175.6s).
These results confirm that our 4-buffer pipeline effectively hides the NVMe write latency, which is the primary bottleneck in TLC-based consumer SSDs.

\noindent\textbf{Circuit Characteristics.}
The performance gain varies by circuit complexity.
\textit{High-complexity circuits} (VQC, VQE, Random), which involve dense gate operations and frequent state vector updates, benefit most from the pipeline.
For VQC at 34 qubits, \EdgeQuantum saves nearly 10 minutes of runtime (2069s $\to$ 1483s).
This is because the heavy GPU computation (approx. 220ms per chunk) provides ample time window to hide the slow NVMe write operations (approx. 450ms) via the 4-buffer overlap.
In contrast, \textit{Low-complexity circuits} (GHZ) show a stable but lower speedup of $\sim$1.21$\times$.
Since GHZ involves very few gates per layer, the GPU compute time is too short to fully hide the write latency, yet \EdgeQuantum still outperforms the blocking baseline by avoiding read/write contention.

\subsection{Extreme Scalability: 35--39 Qubits}
\label{sec:extreme_scale}

To demonstrate the capability of \EdgeQuantum beyond the physical limits of the SSD, we evaluate the system with LZ4 compression enabled for 35--39 qubits.
This regime corresponds to logical state vector sizes of 256\,GB to 4\,TB, which is up to \textbf{512$\times$} larger than the device RAM and \textbf{16$\times$} larger than the SSD capacity.

\noindent\textbf{35-Qubit Transition.}
At 35 qubits (256\,GB), the raw state vector exceeds the 230\,GB usable capacity of the partition.
\EdgeQuantum automatically engages the LZ4 ping-pong mechanism.
Table~\ref{tab:extreme_results} shows the results.
Despite the overhead of compression, \EdgeQuantum maintains practical runtimes.
For example, a 35-qubit VQC simulation (Depth=5) completes in 41.2 minutes (2471s).
Comparing this to the 34-qubit result (1483s), the scaling factor is $1.66\times$, which is better than the theoretical $2\times$ growth of the state space.
This sub-linear scaling is due to the efficiency of LZ4 on sparse state vectors, which reduces the effective I/O bandwidth requirement.

\noindent\textbf{4TB Simulation on Edge.}
The most significant result is the successful execution of 39-qubit simulations.
As shown in Table~\ref{tab:extreme_results}, a 39-qubit Random circuit (Depth=5) completes in 30,535 seconds ($\approx$8.5 hours).
While this is a long duration, it represents a workload that is fundamentally impossible on existing edge simulators.
The linear scaling of runtime from 35 to 39 qubits (approx. $2\times$ per qubit) indicates that the "Ping-Pong" file compression strategy remains stable even as file sizes grow into the terabyte range.
Interestingly, at 37 qubits, we observe a convergence of runtimes for simpler circuits (Random, QV, QSVM) to $\approx$7640s.
This suggests that at the 1\,TB scale, the system hits a bandwidth saturation point where the throughput is limited by the LZ4 compression/decompression rate rather than the raw NVMe speed or GPU compute capability.

\subsection{Fidelity and Stability}
Since \EdgeQuantum partitions the state vector and relies on frequent I/O, ensuring numerical stability is critical.
We verified the fidelity of the final state vector against small-scale \textit{cuQuantum} CPU simulations for verifying correctness.
For all circuits up to 30 qubits (where verification is feasible), \EdgeQuantum achieved a fidelity of $1.0 - \epsilon$ (where $\epsilon < 10^{-6}$), confirming that the chunk-based processing and LZ4 compression/decompression cycle preserves numerical precision (single-precision float).
The successful completion of 151 benchmark runs without a single crash or I/O error validates the robustness of the 4-buffer pipeline and the UVM management logic.
