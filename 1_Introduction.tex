\section{Introduction}
\label{sec:intro}

Quantum computers have been advancing rapidly, demonstrating computational advantages over classical computers on specific complex tasks~\cite{arute2019quantum, markov2018quantum}.
Unlike classical bits, quantum computers utilize qubits, which are mathematically expressed as state vectors describing probability amplitudes (e.g., $|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$). Superposition allows qubits to represent 0 and 1 simultaneously, enabling massive parallelism~\cite{miguel2023enhancing, renner2022computational}.
Additionally, entanglement creates unique correlations between qubits, allowing efficient information processing~\cite{graham2022multi, mooney2019entanglement}.
These properties provide potential advantages in areas like quantum cryptography~\cite{bennett1992experimental}, quantum simulations~\cite{childs2018toward}, and quantum machine learning~\cite{biamonte2017quantum}.
However, current practical quantum computers operate as Noisy Intermediate-Scale Quantum (NISQ) devices. These devices struggle to produce reliable results consistently due to high error rates and the lack of fault tolerance~\cite{preskill2018quantum, lau2022nisq}. To overcome these limitations and effectively verify the accuracy of quantum algorithms, classical hardware-based quantum circuit simulation has become an indispensable tool.

To execute these simulations, high-performance frameworks such as Qiskit Aer~\cite{qiskit}, Google Qsim~\cite{qsim}, and NVIDIA cuQuantum~\cite{bayraktar2023cuquantum} are widely used. Because accurate simulation requires managing an exponentially growing state vector, these frameworks are primarily designed for High-Performance Computing (HPC) systems, assuming access to massive memory pools and interconnected GPU clusters. While such centralized architectures provide abundant computational capacity, relying entirely on remote cloud infrastructure introduces critical bottlenecks for real-world deployment.
Specifically, in emerging applications such as Hybrid Quantum-Classical Machine Learning (HQML), algorithms must continuously process raw sensor data collected by Internet of Things (IoT) devices. Offloading this iterative processing to remote datacenters incurs severe network latency and exposes sensitive data to privacy vulnerabilities. To bypass these overheads and ensure real-time responsiveness, it is increasingly necessary to execute quantum circuits and optimize parameters locally on edge devices.

Figure~\ref{intro_motivation_a} illustrates that executing quantum circuit simulations in edge environments encounters a memory wall despite growing demand.
Edge devices operate under strict power limits and have limited physical memory capacity.
Applying server-grade simulation frameworks directly to these devices causes bottlenecks and Out-of-Memory failures. Simulating a 30-qubit circuit requires 16~GB of memory, which exceeds the 8~GB physical limit of standard edge systems such as the NVIDIA Jetson Orin Nano. Simulating 39 qubits requires a 4~TB state vector footprint, which makes local execution infeasible. Despite this memory barrier, enabling local execution is essential.
As shown in Figure~\ref{intro_motivation_b}, relying on remote cloud services for iterative algorithms like the Variational Quantum Eigensolver (VQE) incurs network overhead.
Cloud execution repeatedly exchanges parameter updates and measurement results over the network. This communication loop increases total execution time due to round-trip latency.
While local CPU simulation avoids network dependency, it remains computationally slow.
In contrast, \EdgeQuantum processes the feedback loop internally.
This on-device execution reduces iteration latency to 9.2~ms, outperforming both cloud and CPU baselines.

Many previous studies, as summarized in Table~\ref{intro_table}, optimize quantum circuit simulation for datacenter or HPC environments.
Frameworks such as Qiskit Aer~\cite{qiskit} and PennyLane~\cite{bergholm2018pennylane} target workstations and assume the state vector fits in physical RAM.
HPC simulators such as HyQuas~\cite{zhang2021hyquas}, Atlas~\cite{xu2024atlas}, ScaleQsim~\cite{10.1145/3771577}, and SnuQS~\cite{park2022snuqs} use multi-node execution to scale memory capacity, but rely on high-bandwidth interconnects such as NVLink and InfiniBand and large GPU clusters.
Other works, such as cuQuantum~\cite{bayraktar2023cuquantum} and BMQSim~\cite{zhang2025bmqsim} use tiered memory or data compression but target server-grade GPUs with dedicated VRAM.
These approaches do not consider the hardware characteristics of edge systems, including limited NVMe bandwidth and shared memory constraints in Unified Virtual Memory (UVM) architectures. Deploying these frameworks on edge devices leads to Out-of-Memory failures or performance degradation due to I/O contention.

To address these limitations, we propose \EdgeQuantum, a scalable full state-vector simulation framework for resource-constrained edge devices.
\EdgeQuantum adopts a tiered-memory architecture that separates logical state vector management from physical memory residency.
It utilizes commodity NVMe storage as the backing store and employs a 4-buffer Unified Virtual Memory (UVM) pipeline to manage data movement.
This pipeline avoids NVMe contention by serializing read and write operations, overlapping flash writes with GPU computation.
Additionally, \EdgeQuantum integrates an LZ4 ping-pong compression strategy to expand storage capacity. This design ensures that data is resident in the GPU on demand while masking storage I/O latency. The goal of \EdgeQuantum is to 1) enable the simulation of large-scale quantum circuits beyond physical memory limits, 2) minimize the performance impact of storage latency, and 3) maximize effective storage capacity.
To achieve these goals, \EdgeQuantum 1) partitions the state vector across VRAM, DRAM, and NVMe, 2) employs an asynchronous pipeline to overlap computation with data movement, and 3) applies LZ4 compression to achieve a 242.7$\times$ storage reduction.
Our evaluation on an 8~GB Jetson Orin Nano demonstrates that \EdgeQuantum successfully performs a 39-qubit simulation (4~TB state vector).
This achieves a 512$\times$ capacity expansion over the physical memory limit.
We open-source the code of \EdgeQuantum at \url{https://github.com/sunggonkim/IoTJ-EdgeQuantum}.
