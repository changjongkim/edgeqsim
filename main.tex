\title{\edgeQsim: Quantum Circuit Simulation Framework for Resource-Constrained Edge Devices}


\maketitle
\begin{abstract}
Quantum circuit simulation is a critical workload for algorithm verification and error characterization, yet it remains predominantly confined to high-performance computing (HPC) and cloud infrastructure due to the exponential memory requirements of state vectors.
On resource-constrained edge platforms, such as the NVIDIA Jetson, standard simulators face a hard "Memory Wall," capping feasible simulations at approximately 29 qubits on an 8~GB device.
While swap-based techniques exist for server-grade systems, they fail to address the specific architectural bottlenecks of edge hardware—namely, the profound bandwidth asymmetry of consumer NVMe SSDs and the latency sensitivity of shared memory architectures.
To bridge this gap, we present \EdgeQuantum, the first tiered-memory quantum circuit simulator specifically architected for embedded edge devices.
\EdgeQuantum introduces a novel 4-buffer asynchronous pipeline that explicitly serializes read/write operations to avoid NVMe contention while overlapping slow TLC flash writes with GPU computation.
Furthermore, it implements a ping-pong LZ4 compression scheme that leverages idle CPU cycles to dynamically expand effective storage capacity.
Our evaluation on an 8~GB NVIDIA Jetson Orin Nano demonstrates that \EdgeQuantum enables the simulation of up to 39 qubits ($\approx$4~TB state vector)—effectively expanding the device's memory capacity by $512\times$.
This capability allows edge devices to verify large-scale circuits, such as QFT and VQC, which were previously executable only on cloud-class resources.
\end{abstract}
